# BNBU 体育成绩管理 Web 联调检查清单

日期：2026-06-12

## 0. 启动与环境

- [ ] 后端本地服务可访问，例如 `http://127.0.0.1:8080/api`
- [ ] CORS 允许 Web 预览来源访问
- [ ] CORS 正式环境不使用 `*`，只允许正式 Web 域名和明确的本地联调来源
- [ ] 登录接口返回 token、用户、默认路由
- [ ] 所有受保护接口未登录时返回 `AUTH_REQUIRED`
- [ ] 当前角色访问越权接口时返回 `FORBIDDEN_ROLE`
- [ ] Web 预览使用 `node frontend/preview-server.cjs` 启动，响应头包含 CSP、X-Frame-Options、X-Content-Type-Options、Referrer-Policy

## 1. 老师端 P0

- [ ] 老师登录后只返回自己可管理课程
- [ ] 切换课程后，名单、任务、异常、成绩、导出预检互相隔离
- [ ] 课程工作台指标与课程名单实际状态一致
- [ ] CSV 名单预检能返回每行 `valid` 与 `status`
- [ ] CSV 名单确认导入只写入通过校验的行
- [ ] 课程任务支持创建、发布、关闭、删除
- [ ] 异常审核支持通过、驳回、补材料
- [ ] 异常通过后只累计一次学时，重复提交不会重复加分
- [ ] 专项考试支持批量保存
- [ ] 签到/平时分支持批量保存
- [ ] 体测原始成绩与折算分支持批量保存
- [ ] 成绩汇总按管理员配置权重实时计算
- [ ] 导出前检查能阻止体测缺失、打卡未满、异常未处理、模板缺失
- [ ] 预检通过后可生成 CSV 或 XLSX
- [ ] 老师提交成绩预检后，管理员端能看到对应归档记录

## 2. 管理端 P0/P1

- [ ] 管理员可查看全校课程健康度
- [ ] 管理员可查看每门课预检问题数量
- [ ] 体育学时规则保存时校验 A 类 + B 类 = 总学时
- [ ] 成绩权重发布时校验四项合计 = 100
- [ ] 导出模板保存时校验必须包含学号、姓名、总分
- [ ] 管理员可配置校队和社团是否可抵扣
- [ ] 管理员可确认组织身份、认可抵扣、取消抵扣
- [ ] 管理员可确认归档、退回清理、提醒老师
- [ ] 管理员退回后老师端能看到退回状态与说明
- [ ] 操作日志能记录所有关键写操作

## 3. 组织负责人端 P0

- [ ] 负责人只能查看自己组织范围内的成员认证记录
- [ ] 单个新增成员后状态为待确认
- [ ] CSV 成员预检能识别组织不存在、重复成员、缺少姓名、缺少学号
- [ ] 确认导入只写入通过校验的行
- [ ] 负责人确认身份后，体育类组织自动设置为可抵扣
- [ ] 非体育类社团不产生体育抵扣
- [ ] 负责人驳回后不影响学生其他运动小时

## 4. 权限与审计

- [ ] 老师不能访问其他老师课程
- [ ] 组织负责人不能查看完整体育成绩
- [ ] 管理员可以查看全校配置与汇总，但敏感导出需记录日志
- [ ] 每次导出记录 actor、courseId、format、time
- [ ] 每次归档确认记录 actor、courseId、issueCount、time
- [ ] 每次组织抵扣变更记录 actor、membershipId、decision、time

## 5. 文件与导出

- [ ] CSV 导入支持 UTF-8 BOM
- [ ] CSV 单元格支持英文逗号和双引号转义
- [ ] 导出 CSV 包含 BOM，避免中文乱码
- [ ] 正式成绩导出字段顺序完全遵循管理员导出模板
- [ ] 导出失败时返回 `EXPORT_BLOCKED` 与问题清单

## 6. 前端联调验收

- [ ] 管理端 `接口联调交付` 页面能打开
- [ ] `导出状态快照 JSON` 可下载
- [ ] `导出路由矩阵 CSV` 可下载
- [ ] `导出接口清单 CSV` 可下载
- [ ] `导出质量安全清单 CSV` 可下载
- [ ] API Base URL 保存后刷新仍保留
- [ ] 健康检查接口失败时能显示明确错误
- [ ] 健康检查接口成功时能显示 HTTP 状态与返回摘要
- [ ] 健康检查显示 URL 策略、请求耗时和重试次数
- [ ] 非 http(s) API URL 被前端阻止
- [ ] 非本地 HTTP API URL 在前端显示正式环境 HTTPS 警告
- [ ] `node frontend/quality-smoke.cjs` 通过，确认安全响应头、路径防护、CORS 与 200 并发烟测

## 7. 稳定性与压测

- [ ] 200 名 Web 用户同时登录并访问老师端/管理端常用页面时，服务端无崩溃
- [ ] 200 名 Web 用户同时执行查询类接口时，P95 响应时间不高于 800ms
- [ ] 200 名 Web 用户混合执行名单导入预检、成绩保存、异常审核、导出预检时，错误率低于 1%
- [ ] 写接口具备幂等键或版本号，重复点击不会重复加学时、重复导入或重复归档
- [ ] 导出接口使用异步任务或后端流式下载，避免大文件阻塞主请求线程
- [ ] 后端连接池、队列和限流参数有明确配置，并可在日志中追踪
- [ ] 压测报告记录测试日期、数据规模、并发用户数、P50/P95/P99、错误率和瓶颈结论

## 8. 网络安全

- [ ] 登录态使用安全 token 策略，正式环境 Cookie 需 `HttpOnly`、`Secure`、`SameSite`
- [ ] 所有写接口校验 CSRF 或使用等效的 token 防护策略
- [ ] 所有输入字段服务端校验长度、类型、枚举值和权限范围
- [ ] CSV 上传限制文件大小、行数、扩展名和 MIME，并在服务端解析时防公式注入
- [ ] 接口具备基础限流，异常 IP 或账号触发 429 与审计日志
- [ ] 关键接口不在错误信息中暴露堆栈、数据库字段或服务器路径
- [ ] 安全响应头在正式 Web 服务中开启：CSP、X-Frame-Options 或 frame-ancestors、X-Content-Type-Options、Referrer-Policy
- [ ] 审计日志覆盖登录失败、越权访问、导入、成绩修改、异常审核、组织抵扣、导出、归档

## 9. 浏览器兼容

- [ ] Chrome 最近两个稳定版本完成主流程验证
- [ ] Edge 最近两个稳定版本完成主流程验证
- [ ] Safari 最近两个稳定版本完成主流程验证
- [ ] Firefox 最近两个稳定版本完成主流程验证
- [ ] 1366px 宽桌面、1440px 宽桌面和 320px 窄屏基础降级布局无内容重叠
- [ ] 下载 CSV 在 Chrome、Safari、Edge 中均能正常生成并包含 UTF-8 BOM
