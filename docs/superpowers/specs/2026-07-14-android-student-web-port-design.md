# Android 学生端 Web 移植设计

## 目标

在现有 `frontend/student/` 原生 JavaScript SPA 上移植 `chchaiai/BNBU-Sports-Android` 的主要学生端功能与 Material 3 视觉语言，交付一个可独立预览、可使用演示数据完整验收、并保留未来真实 API 接入边界的学生 Web 端。

本轮只完成前端与接口适配层，不连接或修改后端数据库，不以正式环境联调作为验收前置条件。

## 设计原则

- 以 Android 仓库当前 `master` 的五栏信息架构、页面内容和视觉 token 为基准。
- 保留现有 Web 的真实凭证上传、视频预览、演示模式、会话与草稿等不冲突能力。
- 手机端追求高保真；平板和桌面端使用相同组件做响应式重排，不机械放大手机布局。
- 不引入 React、Vue 或大型 UI 框架，继续沿用可直接静态部署的 ES Modules SPA。
- 不改教师端、后端数据库、Nginx 或正式服务器配置。

## 信息架构

### 一级导航

登录后的五个一级入口严格按 Android 顺序排列：

1. 首页
2. 课程
3. 打卡
4. 成绩
5. 我的

登录成功、演示模式进入以及未知路由回退时默认进入首页。手机端使用固定底部五栏导航，打卡位于中间并使用加号方框图标；桌面端仍保留底部导航语义，但限制最大宽度并居中显示。

### 二级页面与浮层

- 课程详情、打卡记录详情继续使用 hash 子路由。
- 通知中心采用 Android 式底部抽屉：列表和详情在同一抽屉内切换，支持筛选、单条已读和全部已读。
- 耐力跑成绩换算、免测与免打卡申请、隐私政策作为“我的”下的二级页面。
- 设置能力直接保留在“我的”页面，包含浅色、深色、跟随系统三种主题模式、减少动态效果、演示数据重置和退出登录。

## 页面设计

### 登录

- 使用点阵网格背景、BNBU 品牌锁定、产品标题、说明文字和 Material 3 表单卡片。
- 提供学号/邮箱、密码可见性切换、键盘提交、明确错误状态、加载状态和演示模式入口。
- 桌面端使用品牌介绍与登录卡并列布局；手机端为单列。

### 首页

- 顶部显示学生问候和通知铃铛。
- 学时进度面板展示总完成、课程相关、其他运动和待审核数量。
- 风险面板根据学时缺口、补材料记录和待审核状态生成行动文案与跳转。
- 重点计划展示当前最重要的补齐动作。
- 近期任务展示截止时间、学时和状态。
- 首页内容与 Android `DashboardScreen` 对齐，不复活旧 Web 中已经偏离 Android 的周历和快捷宫格。

### 课程

- 当前学期课程置顶，历史课程折叠显示。
- 课程卡展示 `courseCode / Section`、课程名、教师、学期、学时进度、下一截止时间和修读状态。
- 课程详情展示课程事实、课程相关进度、本教学班任务和相关记录。
- 无课程、无任务和无记录都有明确空状态。

### 打卡

- 保留“任务 / 提交 / 记录”三个分段页签。
- 任务支持状态筛选、任务详情和带入提交表单。
- 提交表单包含学时步进器、运动项目可视选择器、其他运动名称、补充说明、凭证选择、预览、删除、上传进度、失败重试和二次确认。
- 支持草稿保存、恢复、清除，以及从驳回或需补材料记录发起补交。
- 记录支持状态筛选、首张凭证缩略图、`+N` 数量、视频播放标识和详情中的完整媒体预览。
- 本轮演示模式完成全流程；真实 API 调用继续通过现有 Adapter，不要求数据库联调。

### 成绩

- 展示总分预估、四项权重卡、透明计算公式、缺失项风险和数据来源。
- 权重固定为打卡 25%、专项考试 30%、平时表现 20%、体测 25%。
- 保持只读，不允许学生修改教师录入成绩。

### 我的

- 展示学生身份、学院、年级和学号。
- 增加 Android 已有的任课教师信息面板。
- 展示校队/社团认证、有效期和抵扣状态。
- 提供通知、耐力跑换算、免测与免打卡、隐私政策入口。
- 设置区支持主题模式、减少动态效果、缓存/同步状态、演示数据重置和退出登录。

### 耐力跑、免测与隐私

- 耐力跑按性别和年级选择 800m/1000m 规则，显示输入校验、分数、等级、规则来源和 API 失败时的本地换算结果。
- 免测页面包含申请列表、详情、新申请与补交证明，沿用统一凭证选择和上传规则。
- 隐私政策覆盖采集数据、使用目的、保存与安全、用户权利、更新和联系方式。

## 视觉系统

### 色彩

视觉 token 对齐 Android Material 3：

- 主色：`#1A73E8`
- 主色容器：`#D3E3FD`
- 强调橙：`#FD7E14`
- 第三强调色：`#00897B`
- 浅色背景：`#F8F9FA`
- 浅色正文：`#202124`
- 深色背景：`#0F172A`
- 深色表面：`#1E2433`
- 错误色：`#D93025`

全部颜色由 CSS 自定义属性提供，并按 `data-theme` 与 `prefers-color-scheme` 切换。

### 字体、形状与层级

- 字体使用浏览器可稳定回退的中文无衬线栈，字号与字重按 Material 3 层级重设。
- 控件圆角 8px、卡片 12px、大型容器 16px、抽屉 28px。
- 卡片使用轻微色调高程，不使用沉重阴影或大面积渐变。
- 页面背景使用低对比度 42px 点阵网格，呼应 Android `GridBackground`。
- 使用真实 SVG 图标或统一的内联图标组件，不继续使用字符符号作为导航图标。
- 页面切换、抽屉、进度与选中状态使用克制动效；减少动态效果时全部降级。

## 响应式布局

- `<= 600px`：单列手机布局，固定五栏底部导航，触控目标不小于 44px。
- `601px–900px`：单列或局部双列，课程、成绩和首页指标可使用两列。
- `> 900px`：内容最大宽度约 1120px，首页、课程、成绩和个人信息使用双栏或多栏；表单主体限制在 680px 左右。
- 全部断点保证无页面级横向滚动，固定导航不遮挡最后一项内容，并适配 `safe-area-inset-bottom`。

## 状态与数据流

- `createStore` 继续作为唯一客户端状态源；新增 UI 偏好和通知抽屉状态，不把临时 File 对象持久化。
- `createStudentApi` 保持真实接口边界；`createDemoApi` 提供完整验收数据和模拟提交。
- 演示模式明确标识为未连接后端，但不削弱表单校验、上传状态或页面交互。
- 页面渲染函数保持纯函数风格；事件与副作用继续由 `createStudentApp` 统一编排。
- 主题、减少动态效果、会话和草稿写入本地存储；对象 URL 在删除、提交或卸载时释放。

## 错误与降级

- 登录、数据加载、上传、记录提交、耐力跑换算和免测提交均显示具体错误与重试入口。
- API 数据不可用时保留当前页面并显示缓存/演示状态，不静默伪装为正式数据。
- Token 失效时清理会话并返回登录。
- 媒体 URL 继续执行白名单校验；加载失败时显示运动项目占位。
- 主题和动效偏好在浏览器不支持相关能力时回退到浅色与无额外动画。

## 测试与验收

### 自动化

- 使用 Node 内置测试扩展现有 `student-test.mjs`，覆盖路由、五栏导航、首页内容、主题、通知抽屉、历史课程、设置、免测补交和既有上传行为。
- 扩展 `student-smoke.cjs`，在手机、平板和桌面视口逐页检查主入口、抽屉、二级页面、表单和横向溢出。
- 保留并运行 Web、学生端和 API 现有测试，证明教师端与接口适配没有回归。

### 视觉验收

- 生成手机首页、手机打卡、手机我的、平板课程和桌面首页/打卡截图。
- 对照 Android 源码检查导航顺序、主要文案、信息层级、主题 token、圆角与空状态。
- 分别检查浅色、深色和跟随系统模式，以及减少动态效果设置。

### 完成交付标准

- `/student/index.html` 可独立启动并刷新。
- 无需数据库即可通过演示模式完成登录、首页浏览、课程浏览、打卡提交、记录查看、成绩查看、通知处理、耐力跑换算、免测申请/补交、主题切换和退出登录。
- 手机、平板、桌面均无页面级横向溢出或被底部导航遮挡的主操作。
- 正式 API Adapter 与上传边界保留，但本轮不声明已完成生产数据库联调。
