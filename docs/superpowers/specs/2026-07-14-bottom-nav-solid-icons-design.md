# 学生端底栏实心图标设计

## 目标

将学生端底部导航中“课程、打卡、成绩、我的”四枚图标统一为用户参考图中的实心、紧凑风格。首页继续使用现有 BNBU 校徽，不改变导航结构、文字、点击区域或路由行为。

## 视觉规范

- 四枚图标始终使用实心造型，激活态和未激活态不切换轮廓版本。
- 统一使用 `24 × 24` SVG 画布、`currentColor` 填充和接近 `18–20px` 的视觉主体尺寸。
- 未激活态沿用底栏深灰色；激活态沿用品牌蓝，并继续使用现有 `44 × 28px` 浅蓝胶囊背景。
- 图标的视觉重心必须位于画布中心；四枚图标在相同尺寸下具有接近的黑度和占用面积。
- 首页校徽及其激活态保持不变。

## 图标造型

1. **课程**：实心展开书本。左右书页具有轻微外展轮廓，中间保留清晰的竖向书脊负空间。
2. **打卡**：实心圆角方块，中心使用背景色镂空“＋”，与参考图中的 add-box 造型一致。
3. **成绩**：三根不同高度的实心柱状条，底部对齐，柱宽和间距一致。
4. **我的**：实心圆形头像底板，头部与肩部区域使用负空间形成清晰人物剪影。

## 实现边界

- 在现有 `frontend/student/core/icons.js` 中为四枚底栏图标增加实心 SVG 定义和相应渲染属性。
- 其他图标（通知、返回、关闭、刷新、月亮、盾牌）继续使用现有线框渲染，不受本次修改影响。
- `renderBottomNav()` 继续通过现有 `NAV_ITEMS` 和 `icon()` 接口渲染，不增加外部图标字体、图片请求或第三方依赖。
- CSS 仅补充实心图标所需的尺寸/状态规则，不改变底栏高度和导航按钮点击范围。

## 无障碍与交互

- SVG 保持 `aria-hidden="true"`，按钮的可访问名称继续由“首页、课程、打卡、成绩、我的”文字提供。
- 键盘焦点样式、`aria-current="page"` 和路由点击行为保持现状。
- 深色模式下图标继续继承主题文字色和品牌主色，保证对比度。

## 验收标准

1. 首页校徽与当前版本一致。
2. 课程、打卡、成绩、我的均显示为实心图标，造型与参考图对应。
3. 四枚图标在激活和未激活状态下视觉重量一致；激活态仅通过颜色、胶囊背景和文字字重区分。
4. 390×844、768×1024、1440×900 三种视口无水平溢出，底栏高度和点击区域不缩小。
5. 学生端单元测试、静态烟测、Playwright 浏览器验收和整站自测全部通过。
