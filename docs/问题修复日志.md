## #001 — 登录后页面无限刷新导致按钮无法点击

**日期**：2026-07-05  
**影响范围**：dev 环境 (`123.207.5.70:3335`) 全部角色  
**严重程度**：P0 — 功能完全不可用

---

### 问题表现

1. 用户可以正常登录，登录后能看到工作台页面
2. 工作台页面上**所有按钮都无法点击**（侧边栏导航、操作按钮等均无响应）
3. 页面给人一种"**一直在刷新**"的感觉，DOM 不断抖动
4. 浏览器 Console 中反复出现 `GET /api/teacher/courses/gepe/dashboard 401` 或 200 的网络请求

### 根因分析

**位置**：`frontend/app.js` 中 `renderRouteUnsafe()` 函数

`renderRouteUnsafe()` 中有 7 条路由在处理时采用了以下模式：

```js
if (state.route === "teacher-dashboard") {
    fetchCourseDashboard(state.courseId).then(() => renderRoute());
    content = renderDashboard();
}
```

**问题链路**：

```
renderRoute()
  → renderRouteUnsafe()
    → fetchCourseDashboard()        // 发起 API 请求
    → renderDashboard()             // 渲染页面
  → renderLayout()                  // 替换 app.innerHTML
  → saveState()

  // API 请求完成...
  .then(() => renderRoute())        // 再次调用 renderRoute
    → renderRouteUnsafe()
      → fetchCourseDashboard()      // 又发起 API 请求（无缓存检查）
      → renderDashboard()
    → renderLayout()                // 再次替换 app.innerHTML（按钮被销毁重建）
    → saveState()

  // 无限循环 ↑
```

**根本原因**：`fetchCourseDashboard` 没有缓存检查机制，每次调用都会发起新的 API 请求。请求完成后通过 `.then(() => renderRoute())` 触发重新渲染，重新渲染又调用同一个 fetch 函数，形成**无限渲染循环**。

每次 `renderLayout()` 调用 `app.innerHTML = ...` 都会**销毁并重建整个 DOM**，导致：
- 按钮元素被销毁，点击事件传播链路断裂
- 用户无法与任何元素交互
- 页面持续抖动

受影响的路由共 7 条：

| 路由 | fetch 函数 | 缓存字段 |
|---|---|---|
| `teacher-dashboard` | `fetchCourseDashboard()` | `state.dashboardCache[courseId]` |
| `teacher-progress` | `fetchCourseGrades()` | `state.gradeData` |
| `teacher-review` | `fetchCourseReviews()` | `state.reviews` |
| `teacher-grades` | `fetchCourseGrades()` | `state.gradeData` |
| `admin-dashboard` | `fetchAdminOverview()` | `state.adminOverview` |
| `admin-overview` | `fetchAdminOverview()` | `state.adminOverview` |
| `admin-logs` | `fetchAdminLogs()` | `state.logs` |

---

### 解决方案

在每个路由的 fetch 调用前增加**缓存检查**：如果数据已经加载过（缓存非空），跳过 API 请求，只做纯渲染。

**修改前**（以 teacher-dashboard 为例）：

```js
if (state.route === "teacher-dashboard") {
    fetchCourseDashboard(state.courseId).then(() => renderRoute());
    content = renderDashboard();
}
```

**修改后**：

```js
if (state.route === "teacher-dashboard") {
    state.dashboardCache = state.dashboardCache || {};
    if (!state.dashboardCache[state.courseId]) {
        fetchCourseDashboard(state.courseId).then(() => renderRoute());
    }
    content = renderDashboard();
}
```

**执行流程**（修复后）：

```
首次进入：
  renderRoute() → dashboardCache 为空 → 发起 API 请求 → 渲染骨架
  → API 返回 → 缓存数据 → .then(renderRoute) → 缓存命中 → 纯渲染（无请求）

后续访问：
  renderRoute() → 缓存命中 → 跳过 API → 纯渲染
```

其余 6 条路由采用相同模式修复：

```js
// teacher-progress / teacher-grades
if (!state.gradeData) { fetchCourseGrades(courseId).then(() => renderRoute()); }

// teacher-review
if (!state.reviews || !state.reviews.length) { fetchCourseReviews(courseId).then(() => renderRoute()); }

// admin-dashboard / admin-overview
if (!state.adminOverview) { fetchAdminOverview().then(() => renderRoute()); }

// admin-logs
if (!state.logs || !state.logs.length) { fetchAdminLogs().then(() => renderRoute()); }
```

---

### 修复成果

| 指标 | 修复前 | 修复后 |
|---|---|---|
| 页面渲染次数 | 无限循环（~20次/秒） | 1-2 次（首屏骨架 + 数据回填） |
| 按钮可点击 | ❌ 完全不可用 | ✅ 正常响应 |
| 侧边栏导航 | ❌ 无法切换路由 | ✅ 正常切换 |
| API 请求数（dashboard） | 无限 | 首次 1 次 |
| DOM 稳定性 | 持续销毁重建 | 仅在数据到达时更新一次 |

---

### 后续建议

1. **统一 API 加载守卫**：建议所有 async fetch 函数统一使用 `shouldStartApiLoad(key)` 模式，避免类似死循环在其他路由中复现
2. **渲染防抖**：在 `renderRoute()` 外层增加简单的防抖（如 requestAnimationFrame），同一帧内多次调用合并为一次
3. **e2e 冒烟测试**：覆盖「登录 → 工作台 → 点击导航按钮」路径，确保交互可用
