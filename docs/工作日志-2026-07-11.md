## 二、双前端分流机制

| 角色 | 入口 | 说明 |
|------|------|------|
| 体育任课老师 | 登录后自动跳转 | `frontend/teacher/` Material UI |
| 体育部管理员 | 登录后留在主 SPA | `frontend/index.html` + `app.js` |
| 校队 / 社团负责人 | 登录后留在主 SPA | 同上 |
| 演示模式 | `teacher/index.html?demo=1` 或登录页快速演示 | 无需后端 token |

**关键文件：**

- `frontend/app.js` — `redirectToTeacherApp()`、`handleLogin()`、`teacherAppUrl`
- `frontend/teacher/auth-bridge.js` — 与主应用共享 `sessionStorage: bnbuAuthSession`

## 七、已知遗留与建议

1. **免测审核**：Material 教师端主要在「审核工作台」部分 tab，与原 `teacher-exemptions` 独立页不完全等价。
2. **打卡进度**：由「打卡全览 / 历史记录」部分覆盖，非原 `teacher-progress` 一一对应。
3. **敏感信息**：`database/run-seed.cjs` 含硬编码远程 MySQL 凭据，生产环境建议改为环境变量并轮换密码。
4. **样式层叠**：管理端仍加载 `styles-campus-blue.css`，由 `app-material.css` 覆盖；后续可考虑合并精简。
