# BNBU 学生体育 Web 端验收说明

## 1. 交付入口

- 本地启动：在 `BNBU-web` 根目录执行 `npm run preview`
- 学生端：`http://127.0.0.1:4174/student/index.html`
- 产品文档：`docs/学生体育Web端产品文档_v1.1.md`
- 演示方式：登录页点击“进入演示模式”，无需数据库
- 真实模式：使用学生账号登录，同源请求 `/api/*`

## 2. 功能验收

| 范围 | 可验收成果 |
|---|---|
| 打卡（默认入口） | 独立居中的核心打卡胶囊；任务、提交、记录、草稿、被驳回记录补交材料；图片/视频选择、校验、预览、上传状态和确认提交 |
| 打卡记录 | 列表显示首个有效凭证缩略图、多凭证数量和视频标识；详情预览全部图片或视频 |
| 课程 | 课程列表、课程详情、课程任务和相关打卡记录 |
| 成绩 | 总分预估、四项权重、计算公式、缺失项和数据来源 |
| 我的 | 学生身份、校队/社团抵扣、通知中心、免测申请、设置和退出登录 |
| 二级功能 | 耐力跑成绩换算；800m/1000m 免测申请与申请记录 |
| 接口 | 学生登录/数据聚合、课程详情、成绩、打卡、凭证上传、通知已读、免测申请等契约；断网时可使用明确标识的演示模式 |

## 4. 自动化验证

```bash
npm run test:student
npm run test:web
npm run test:api
$env:STUDENT_WEB_URL='http://127.0.0.1:4174/student/index.html'; npm run smoke:student
```

学生端测试覆盖领域工具、草稿/持久化、上传规则、成绩计算、打卡状态、API 映射和静态入口。API 测试覆盖学生接口与混合媒体上传限制。

## 5. 手工交互结果

- 单条/全部通知可标为已读。
- 耐力跑 `3:45` 可换算为 88 分（良好）。
- 800m 免测申请可提交并立即出现在申请记录中。
- 未上传凭证时提交打卡会显示“请至少上传 1 个有效凭证”。
- 首页模块已移除；旧 `#/home` 地址自动回到打卡。
- 打卡胶囊在底部三项导航上方居中，课程、成绩、我的等宽排列。
- 记录列表可显示图片缩略图、`+N` 数量和无凭证占位。

## 6. 联调边界

当前成果可使用演示数据完整验收前端流程。真实模式已连接仓库内 API 契约，但正式环境仍需使用实际学生账号、MySQL 数据和 Nginx `/api/`、`/uploads/` 配置进行部署联调。
